## IMMEDIATELY (This Sprint)

### 1. Fix WCAG AA Color Contrast Failures (53 elements)
- **Category:** a11y
- **Impact:** Critical — legal risk (WCAG 2.1 AA failure)
- **Effort:** M
- **Files:** All components using `text-black-400`, `text-black-500`, `text-gold-500/70`, badge classes
- **Fix:**
  - `text-black-500` on dark bg: contrast 3.46:1 → needs 4.5:1. Change to `text-black-400` or lighter
  - `text-black-400` on `bg-black-700`: contrast 4.03:1 → bump to `text-black-300` (#7A9AB8 → ~5.3:1)
  - `text-gold-500/70`: contrast 4.39:1 → remove opacity or use `text-gold-400`
  - `badge-recommend` (white on emerald): 2.53:1 → use dark text (`text-black-950`) on emerald, or darken emerald to `#059669`
  - `text-[10px]` labels: increase to minimum 12px and fix contrast
  - Run `npx @axe-core/cli http://localhost:5173` after fixes to verify

### 2. Add Accessible Name to Sort Dropdown
- **Category:** a11y
- **Impact:** Critical — WCAG 2.1 A failure (4.1.2 Name, Role, Value)
- **Effort:** S
- **Files:** Component rendering the `<select>` in the filter bar (likely `src/components/layout/FilterBar.tsx` or `src/components/filters/AdvancedSortPanel.tsx`)
- **Fix:** Add `aria-label="Sort screenplays by"` to the `<select>` element

### 3. Fix React Duplicate Key Warnings
- **Category:** engineering
- **Impact:** High — can cause rendering bugs, duplicated/omitted children
- **Effort:** S
- **Files:** `src/components/screenplay/ScreenplayGrid.tsx` (likely), wherever the screenplay list is rendered
- **Fix:** The key `matadero-v4-1` appears multiple times in the data. Ensure each screenplay has a unique key — if IDs aren't unique, append the array index: `key={\`${sp.id}-${index}\`}`

### 4. Fix Mobile Responsive Layout
- **Category:** design
- **Impact:** High — filter bar overflows/wraps poorly on 375px
- **Effort:** M
- **Files:** `src/components/layout/FilterBar.tsx`, `src/components/layout/Header.tsx`
- **Fix:**
  - Filter bar buttons should collapse into a hamburger menu or horizontal scroll on mobile
  - Add breakpoints below 640px for filter bar layout
  - Currently only 4 `@media` queries total — needs more responsive breakpoints
  - Touch targets on mobile should be minimum 44x44px

### 5. Add ErrorBoundary Coverage for Remaining Routes
- **Category:** engineering
- **Impact:** High — crashes without boundaries = data loss
- **Effort:** S
- **Files:** `src/App.tsx`, individual page components
- **Fix:** ErrorBoundary is used in App.tsx and ScreenplayGrid — verify all lazy-loaded routes are wrapped. The SettingsPage, SharedViewPage, and comparison flows should each have boundaries.
