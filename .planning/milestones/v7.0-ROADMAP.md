## Milestone Summary

**Key Decisions:**
- Always-on checkboxes (not mode toggle) — producer selects often enough that a mode toggle becomes friction
- Sticky bottom action bar — keeps filters visible while selecting; standard pattern
- Streamlined upload modal (not full panel) — selection already done in grid
- Visible-but-disabled buttons with tooltips (never hide actions)
- No global selection cap; per-action limits at the button level
- One dropzone per screenplay (no filename auto-matching for initial assignment)
- Row-based virtualization with @tanstack/react-virtual (not cell-based)
- Set-based Zustand selection store for O(1) operations
- uploadBytesResumable for progress-tracked Firebase uploads

**Issues Resolved:**
- Coverage PDF cover page score/verdict spacing fixed
- Virtual scrolling library evaluation — @tanstack/react-virtual selected
- Fire-once animation gating moved to module-level Set (persists across re-mounts)
- BackToTopButton offset when BulkActionBar visible

**Issues Deferred:**
- Keyboard shortcuts for selection (Shift+click range, Cmd+A) — BULK-E3 in backlog
- Bulk share token management — BULK-E1 in backlog
- Bulk delete with confirmation — BULK-E2 in backlog

**Technical Debt Incurred:**
- MATCH_THRESHOLD = 50 is a tunable constant — may need adjustment with real filenames
- Overscan of 3 rows is a performance trade-off (smooth scrolling vs DOM count)
- Virtual scrolling keyboard navigation deferred (CSS Grid column detection removed)
