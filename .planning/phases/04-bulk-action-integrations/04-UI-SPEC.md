| Property | Value |

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px (text-sm) | 400 (normal) | 1.5 (leading-normal) |
| Label | 13px (text-sm) / 11px (text-xs) for uppercase | 500 (medium) | 1.5 (leading-normal) |
| Heading | 18px (text-lg) | 600 (semibold) | 1.25 (leading-tight) |
| Button | 14px (text-sm) | 500 (medium) | 1.5 (leading-normal) |

Phase-specific usage:
- **Modal titles:** 18px semibold, font-heading (Barlow Condensed), text-gold-200
- **Button labels:** 14px medium, default font-body (Satoshi/Inter)
- **Toast messages:** 14px normal, text-white/90
- **Progress text:** 14px medium, text-gold-200 (inline in button during PDF export)
- **Count label:** 14px medium, text-gold-200 (existing BulkActionBar pattern)
- **Tooltip text:** Native `title` attribute (browser default rendering)

### Component Visual Specifications

#### SetCategoryModal

```
Layout:
  - Fixed overlay: z-50, bg-black-950/80, backdrop-blur-sm
  - Panel: max-w-md, glass, border border-gold-500/20, rounded-xl, animate-scale-in
  - Header: px-6 py-4, text-lg font-heading text-gold-200, "Set Category"
  - Body: px-6 py-4
    - Label: text-sm text-black-400, "Choose a category for N screenplays"
    - Dropdown: w-full, glass-input styling, mt-2
      - Options: existing categories from useCategories()
      - No "create new" option (D-02)
  - Footer: px-6 py-4, flex justify-end gap-3
    - Cancel: btn btn-ghost text-sm
    - Apply: btn btn-primary text-sm, "Set Category"
      - Disabled when no category selected
```

Interaction:
- Opens when "Set Category" button clicked in BulkActionBar
- Dropdown populated from `useCategories().categories`
- "Set Category" calls `patchAnalysisField(sourceFile, 'collection', categoryId)` for each selected screenplay
- On success: toast "Category set to {NAME} for {N} screenplays" (success severity, 3s auto-dismiss)
- Selection stays intact after action (D-04)
- Modal closes after successful operation
- Backdrop click closes modal

#### AddToFavoritesModal

```
Layout:
  - Fixed overlay: z-50, bg-black-950/80, backdrop-blur-sm
  - Panel: max-w-md, glass, border border-gold-500/20, rounded-xl, animate-scale-in
  - Header: px-6 py-4, text-lg font-heading text-gold-200, "Add to Favorites"
  - Body: px-6 py-4, space-y-3
    - Quick Favorites row (always first):
      - flex items-center gap-3, p-3, rounded-lg, bg-black-800/50, border border-black-700
      - Star icon (w-5 h-5, text-gold-400)
      - "Quick Favorites" label (text-sm font-medium text-gold-200)
      - Radio/checkbox on right
    - Named lists (if any):
      - Same row pattern per list
      - List name (text-sm text-black-200)
      - Radio/checkbox on right
    - If no named lists: only Quick Favorites shown (no empty state needed)
  - Footer: px-6 py-4, flex justify-end gap-3
    - Cancel: btn btn-ghost text-sm
    - Add: btn btn-primary text-sm, "Add to Favorites"
      - Disabled when no list selected
```

Interaction:
- Opens when "Favorites" button clicked in BulkActionBar
- Quick Favorites always listed as first option
- User selects one list (radio selection, not multi-list)
- "Add to Favorites" loops `favoritesStore.addToList(listId, id)` or `toggleQuickFavorite(id)` for each selected screenplay
- On success: toast "Added {N} to {list name}" (success severity, 3s auto-dismiss)
- Selection stays intact after action (D-04)
- Modal closes after successful operation
- Backdrop click closes modal

## Interaction Contracts

### Post-Action Behavior (D-04)
Selection stays intact after every bulk action. User can chain multiple actions on the same set (e.g., set category, then export CSV, then add to favorites) without re-selecting. The "Clear selection" (X button) and "Deselect All" are always available to manually clear.

### Z-Index Layering (D-06)
| Layer | z-index | Components |
|-------|---------|------------|
| BulkActionBar | z-40 | Sticky bottom bar -- stays visible behind modals |
| Modals | z-50 | SetCategoryModal, AddToFavoritesModal, ComparisonModal |
| Toasts | z-50 | ToastContainer (positioned at bottom-center, above bar) |

### Modal Dismissal
All modals close on:
- Backdrop click
- Cancel button click
- Successful action completion
- Escape key (if existing modal pattern supports it)

### Double-Click Prevention (D-13)
During PDF export: Export PDF button shows progress text and is non-interactive (pointer-events-none). Prevents re-triggering. Reverts after completion.

### Compare Guard (D-08, D-09)
Compare button uses conditional disable: `disabled={count < 2 || count > 3}`. When disabled, native `title` attribute shows "Select 2-3 to compare". When enabled (exactly 2 or 3 selected), click immediately opens ComparisonModal (D-10) -- no confirmation gate.

## Accessibility Contract

| Element | Requirement |
|---------|-------------|
| Modal focus trap | Focus should be trapped within modal when open (follow existing modal pattern) |
| Button disabled state | `disabled` attribute + `title` for tooltip + `disabled:opacity-40 disabled:cursor-not-allowed` |
| Toast announcements | `role="alert" aria-live="assertive"` (existing pattern in ToastContainer) |
| Modal backdrop | Clickable to dismiss; not focusable |
| Dropdown (SetCategoryModal) | Native `<select>` element for built-in keyboard navigation |
| Radio buttons (AddToFavoritesModal) | Native `<input type="radio">` with `<label>` association |
| Color contrast | Success green (emerald-500 #10B981) on dark background meets WCAG AA for large text; toast message text is text-white/90 on bg-black/70 |
