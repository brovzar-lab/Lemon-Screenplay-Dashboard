<objective>
Fix the whole-store Zustand subscription in `useFilteredScreenplays.ts`. Currently `useFilterStore()` with no selector subscribes to the entire store object, causing the `filteredScreenplays` useMemo to re-run on every Zustand state change — even changes to fields unrelated to filtering (e.g., ephemeral UI state if the store grows). Replace with `useShallow(s => s)` so only genuine filter field changes invalidate the memo.

Purpose: PERF-02 — filter and sort operations must not cause unnecessary recomputation. Surgical one-file change, no component changes.
Output: Modified useFilteredScreenplays.ts with useShallow import and selector fix.
</objective>
