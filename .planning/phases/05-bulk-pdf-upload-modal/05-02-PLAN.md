    **Component-local state (D-09: ephemeral, not Zustand):**
    - `rowStates: Record<string, RowUploadState>` -- per-screenplay upload state, initialized to `{ status: 'idle' }` for each missing-PDF screenplay
    - `dragActiveId: string | null` -- which row is currently a drag target (for highlight)
    - `batchDragActive: boolean` -- whether files are being dragged over the batch zone
    - `batchError: string | null` -- error message for batch zone (unmatched files)
    - `retriedIds: Set<string>` -- tracks which screenplays have been auto-retried (D-07: exactly once)

    **Data derivation:**
    - Get `selectedIds` from `useSelectionStore`
    - Get `allScreenplays` from `useScreenplays()`
    - Filter to `missingPdfScreenplays = allScreenplays.filter(sp => selectedIds.has(sp.id) && !sp.hasPdf)`
    - Compute `alreadyAttachedCount = selectedIds.size - missingPdfScreenplays.length` for info note (D-13)
    - Compute live summary counts from `rowStates`: `doneCount`, `errorCount`, `remainingCount`

    **Upload function (per D-04: immediate, per D-07: auto-retry once):**
    Use `uploadBytesResumable` from `firebase/storage` (NOT the existing `uploadScreenplayPdf` which uses `uploadBytes` without progress):
    ```typescript
    import { ref, uploadBytesResumable, getDownloadURL } from 'firebase/storage';
    import { storage } from '@/lib/firebase';
    import { buildStoragePath } from '@/components/settings/pdfUploadPanel.helpers';
    import { patchAnalysisField } from '@/lib/analysisStore';
    import { usePdfStatusStore } from '@/stores/pdfStatusStore';

    const startUpload = (file: File, screenplay: Screenplay) => {
      setRowStates(prev => ({ ...prev, [screenplay.id]: { status: 'uploading', progress: 0 } }));
      const path = buildStoragePath(screenplay);
      const storageRef = ref(storage, path);
      const uploadTask = uploadBytesResumable(storageRef, file, {
        contentType: 'application/pdf',
        customMetadata: {
          originalFilename: file.name,
          category: screenplay.category || 'OTHER',
          uploadedAt: new Date().toISOString(),
        },
      });
      uploadTask.on('state_changed',
        (snapshot) => {
          const progress = Math.round((snapshot.bytesTransferred / snapshot.totalBytes) * 100);
          setRowStates(prev => ({ ...prev, [screenplay.id]: { status: 'uploading', progress } }));
        },
        (error) => {
          // Auto-retry once (D-07)
          if (!retriedIds.has(screenplay.id)) {
            setRetriedIds(prev => new Set(prev).add(screenplay.id));
            startUpload(file, screenplay);
          } else {
            setRowStates(prev => ({
              ...prev,
              [screenplay.id]: { status: 'error', message: error.message, file },
            }));
          }
        },
        async () => {
          await patchAnalysisField(screenplay.sourceFile, 'hasPdf', true);
          usePdfStatusStore.getState().setStatus(screenplay.id, 'found');
          queryClient.invalidateQueries({ queryKey: SCREENPLAYS_QUERY_KEY });
          setRowStates(prev => ({ ...prev, [screenplay.id]: { status: 'done' } }));
        }
      );
    };
    ```

    **Per-row drop handler (D-02, D-15):**
    - `e.preventDefault()` + `e.stopPropagation()` (CRITICAL: prevent batch zone from also firing)
    - Check `files.length > 1` -- reject with row-level error "One file per title"
    - Call `validatePdfFile(files[0])` -- show `validationMessage()` as row error on failure
    - Call `startUpload(files[0], screenplay)` on success
    - `onDragOver`: `e.preventDefault()` (CRITICAL: prevents browser from opening file) + set `dragActiveId`
    - `onDragEnter`: `e.preventDefault()` + set `dragActiveId`
    - `onDragLeave`: clear `dragActiveId`

    **Per-row Browse button:**
    - Hidden `<input type="file" accept=".pdf" />` triggered by visible button click
    - On change: validate + startUpload, same as drop handler but for single file

    **Batch drop zone (D-03):**
    - Sits above the row list
    - `onDrop`: `e.preventDefault()`, extract all files, call `matchFilesToScreenplays(files, missingPdfScreenplays)`
    - For each match: call `startUpload(match.file, match.screenplay)`
    - For unmatched files: set `batchError` to `"{N} file(s) could not be matched to any title"` -- clear after 4 seconds with setTimeout
    - `onDragOver`/`onDragEnter`/`onDragLeave`: manage `batchDragActive` state for highlight

    **Stray drop prevention (D-15):**
    - The modal's outer container (inside the glass border) gets `onDrop={e => { e.preventDefault(); }}` and `onDragOver={e => { e.preventDefault(); }}` to prevent the browser from navigating when files are dropped outside specific zones
    - Since rows use `stopPropagation`, only drops outside rows bubble up. The batch zone sits above the rows. Drops landing on the glass container outside both zones are silently prevented.

    **Layout structure (matching glass/gold modal pattern):**
    ```
    fixed z-50 overlay -> backdrop blur -> glass container (max-w-lg)
      Header: "Upload PDFs" title
      Info note (D-13): "2 of 5 selected screenplays already have PDFs attached" (only if alreadyAttachedCount > 0)
      Summary bar (D-10): "Uploaded {done} of {total} -- {remaining} remaining" or "checkmark {done} of {total} PDFs uploaded successfully"
      Batch drop zone (D-03): dashed border area "Drop multiple PDFs here to auto-match by filename"
      Scrollable row list (max-h-[60vh] overflow-y-auto):
        Per-row: title + category tag left, status indicator + Browse button right, ~h-10 per row
        Row states: idle (default), drag-active (gold border), uploading (progress bar), done (green check), error (red + Retry)
      Footer: Done button (always enabled per D-11)
    ```

    **Row rendering per state:**
    - `idle`: Title + category tag, Browse button on right
    - `uploading`: Title + progress bar (Tailwind bg-gold-500/30 with width percentage), progress percentage text
    - `done`: Title + green checkmark icon + "Uploaded" text, filename shown via `middleTruncate(file.name, 30)`
    - `error`: Title + red error message + Retry button (calls `startUpload(rowState.file, screenplay)`)

    **Filename display (D-16):**
    Use `middleTruncate(file.name, 35)` when showing uploaded or error filenames to handle long names like `El_Godin_de_los_Suenos_V4_FINAL_LOCKED.pdf`.

    **No toast on close (D-12):**
    The `onClose` prop is called directly with no toast. The modal already shows all results inline.

    `if (!isOpen) return null;` at top of render (matching SetCategoryModal pattern).
