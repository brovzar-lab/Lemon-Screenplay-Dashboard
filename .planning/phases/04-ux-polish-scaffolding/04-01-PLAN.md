  <files>src/stores/toastStore.ts, src/stores/toastStore.test.ts, src/stores/index.ts</files>
  <behavior>
    - addToast('message') creates a toast with severity 'error' (default), unique id, and createdAt timestamp
    - addToast('message', 'warning') creates a toast with severity 'warning'
    - removeToast(id) removes the toast with that id; is idempotent (no error if id missing)
    - clearToasts() removes all toasts
    - Toast array length is capped at 10 entries (prevents memory leak from rapid errors)
    - Auto-dismiss fires after ~5 seconds (use vi.useFakeTimers to test)
  </behavior>
  <action>
    Create `src/stores/toastStore.ts` as a Zustand store (NO persist middleware — ephemeral like syncStatusStore). Follow the pattern from 04-RESEARCH.md Pattern 1.

    Type definitions:
    - `ToastSeverity = 'error' | 'warning'`
    - `Toast = { id: string; message: string; severity: ToastSeverity; createdAt: number }`
    - `ToastStore = { toasts: Toast[]; addToast: (message: string, severity?: ToastSeverity) => void; removeToast: (id: string) => void; clearToasts: () => void }`

    Constants: `MAX_VISIBLE = 3`, `MAX_STORED = 10`, `AUTO_DISMISS_MS = 5000`

    Key behaviors:
    - `addToast`: Generate id with `Date.now()-Math.random().toString(36).slice(2,7)`. Append to array. Cap at MAX_STORED (keep newest). Schedule `setTimeout` for auto-dismiss at AUTO_DISMISS_MS.
    - `removeToast`: Filter by id (idempotent).
    - `clearToasts`: Set toasts to [].
    - Export MAX_VISIBLE constant for ToastContainer to use.

    Add `export * from './toastStore';` to `src/stores/index.ts`.

    Write tests in `src/stores/toastStore.test.ts`:
    - Test add/remove/clear operations
    - Test default severity is 'error'
    - Test array cap at 10
    - Test auto-dismiss with fake timers
    - Test idempotent remove
  </action>
