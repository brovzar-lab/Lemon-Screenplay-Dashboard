Create `src/components/bulk/BulkShareModal.tsx`:

**Row state type:**
```typescript
type ShareRowStatus = 'pending' | 'generating' | 'done' | 'failed';
interface ShareRow { status: ShareRowStatus; url?: string; }
```

**State:** `const [rows, setRows] = useState<Record<string, ShareRow>>(() => Object.fromEntries(screenplays.map((sp) => [sp.id, { status: 'pending' }])))`

**Props:**
```typescript
interface BulkShareModalProps {

  onClose: () => void;
  screenplays: Screenplay[];
}
```

**Sequential loop:** Use `useRef<boolean>` cancelledRef (not used for BULK-01 — no cancel button — but follow the same ref pattern from RESEARCH.md). Run the loop inside `useEffect(() => { runBulkShare(screenplays); }, [])` — fires once on mount.

**runBulkShare function** (inside component, using the exact pattern from RESEARCH.md Pattern 2):
1. Cache check: `useShareStore.getState().tokens[sp.id]`
2. Firestore check: `await getExistingShareToken(sp.id)` — if found, call `useShareStore.getState().setToken(sp.id, existing)`
3. Create new: `await createShareToken(sp.id, sp, false)` — setToken with result
4. URL construction: `${window.location.origin}/share/${token}` (use `getShareBaseUrl()` from shareService if exported, otherwise construct directly)
5. On catch: setRowStatus to 'failed', continue loop (do not break)

**Retry handler:** Individual function that re-runs the token generation for one screenplay ID. Same cache→Firestore→create logic.

**Copy All behavior:** `urls.join('\n')` where urls = Object.entries(rows).filter(([,r]) => r.status === 'done').map(([,r]) => r.url!)

**Modal header:** "Generate Share Links" title + `{doneCount} of {total} complete` counter that updates as rows resolve.

**Modal width:** `max-w-lg` (wider than standard md to accommodate URL rows with copy buttons).
